LAB1

Lab1: Unix Utilities

前言

本实验是系统编程相关的实验，不涉及操作系统内核。只需要编写一些用户态程序，执行系统调用来达到目标。作为第一个实验，本次实验内容比较简单，主要内容是展示用户如何调用操作系统的接口，即system call，在下一个实验中将详细展示系统调用的工作流程。

本次实验的学习目标是：

  • 学习并理解用户使用shell执行命令时的具体过程；
    • 命令行传入的参数如何被进程解析并使用？
  • 对Unix的进程间通信方式之一pipe有一定的了解；
    • 为什么用户进程
  • 了解Unix中的file description，理解Unix中“一切皆文件”的理念；
  • 初步了解make项目以及Makefile。

提示：

  1. 本实验只需使用user/user.h文件中暴露给用户使用的系统调用接口来完成相关用户态应用程序的编写，无需修改操作系统内核；
  2. 同时需要在Makefile文件中的UPROGS字段中将编写的用户程序加入，这样才可以在操作系统的命令行中使用该应用程序；
  3. 我们编写的用户态程序最终是要以命令行的形式来驱动的，所以如何从argc, argv中提取命令行参数是必须知道的，RTFM。

注意：在做Lab1的时候，一定要分清所谓的用户态程序和系统调用之间的区别，我们在这个实验中写的是应用程序，它要调用同名的系统调用从而实现相关的功能。所有在用户态下可用的系统调用和lib库函数都可在user/user.h头文件中查看得到，同时鼓励学有余力的同学深入了解其相关的实现细节。

前置工作

git checkout lab1_utils     # 切换到lab1_utils分支，对应本实验

Sleep

任务

写一个用户程序sleep，执行系统调用来实现休眠。

提示

  • 观察user/grep.c等文件来学习命令行参数如何传入程序。
  • 将sleep.c添加进Makefile中的UPROGS。（添加完后点击Makefile右上角图标重新加载make项目才能将sleep.c加入索引）
  • 在用户没有给出sleep的参数时打印错误信息。
  • 使用atoi函数将命令行参数由字符串转为整形。
  • 确保main函数调用了exit函数来退出程序。

pingong

写一个程序，使用unix system calls在两个进程间ping-pong一个字节，使用一对pipe，一个pipe对应一个方向，另外一个pipe对应另外一个方向。
