LAB3

Lab3: PV Operation

1. 实验目的

在该操作系统源码中，提供了关于自旋锁用于内核程序代码的并发同步，但是，对于我们自己编写用户代码，并没有提供进程同步机制，这次实验我们在实验2框架基础上继续动手来实现一个简单的信号量机制，从而达到用户程序代码的进程同步，当然，本次实验是建立在该操作系统源码的自旋锁实现上，在本次实验实现信号量机制的过程中，还可以很好的了解该操作系统的自旋锁是如何实现的。

2. 实验内容及要求

为了实现信号量，我们需要增加四个系统调用，分别是sem_create()创建信号量，其参数是信号量的初值（例如互斥量则用1做初值），返回值是信号量的编号，即内核变量sems[]数组的下标。sem_p()则是对指定编号的信号量进行P操作（减一操作、down操作），反之sem_v()则是对指定编号的信号量进行V操作（增一操作、up操作），这两个操作和操作系统理论课堂上讨论的行为一致，都会涉及到进程的睡眠或唤醒操作。sem_free()是撤销一个信号量。

  • int sem_create(int n_sem)：参数n_sem是初值，返回的是信号量的编号，0为出错。
  • int sem_p(int sem_id)：减一操作，减为0时阻塞睡眠，记录到sem.procs[]中，返回值0表示正常，返回值1则出错。
  • int sem_v(int sem_id)：增一操作，增加到0时唤醒队列中的进程，清除sems[id].procs[]对应的进程号，返回值为0表示成功，1表示出错。
  • int sem_ free (int sem_id)：释放指定id的信号量。返回值为0表示成功，-1表示出错

3. 编写信号量测试代码

在编写完相关信号量机制的系统调用之后，请自己独立编写一个用户态程序来对自己所实现的信号量机制的系统调用进行测试，验证其正确性。
